# Resource: mongodbatlas_stream_connection_failover

`mongodbatlas_stream_connection_failover` provides a Stream Failover Connection resource. The resource lets you create, update, delete, and import a failover (regional-alternate) connection for an existing stream connection.

A failover connection shares the primary connection's `connection_name` and is created for one of the workspace's failover regions (see the `failover_regions` argument of `mongodbatlas_stream_workspace`). It is uniquely identified by its region and its computed `id` (the failover connection ID).

## Example Usages

```terraform
resource "mongodbatlas_stream_connection" "example" {
  project_id        = var.project_id
  workspace_name    = var.workspace_name
  connection_name   = "KafkaConnection"
  type              = "Kafka"
  bootstrap_servers = "localhost:9092,localhost:9092"
  authentication = {
    mechanism = "PLAIN"
    username  = var.kafka_username
    password  = var.kafka_password
  }
  security = {
    protocol = "SASL_SSL"
  }
  config = {
    "auto.offset.reset" = "earliest"
  }
}

# A failover connection shares the primary connection's name and is created for one of the
# workspace's failover regions. It carries its own regional connection configuration.
resource "mongodbatlas_stream_connection_failover" "example" {
  project_id        = var.project_id
  workspace_name    = var.workspace_name
  connection_name   = mongodbatlas_stream_connection.example.connection_name
  region            = var.failover_region
  type              = "Kafka"
  bootstrap_servers = var.failover_bootstrap_servers
  authentication = {
    mechanism = "PLAIN"
    username  = var.kafka_username
    password  = var.kafka_password
  }
  security = {
    protocol = "SASL_SSL"
  }
  config = {
    "auto.offset.reset" = "earliest"
  }
}
```

## Import
You can import a stream failover connection using the workspace name, project ID, connection name, and failover connection ID, in the format `WORKSPACE_NAME-PROJECT_ID-CONNECTION_NAME-FAILOVER_CONNECTION_ID`. For example:
```
$ terraform import mongodbatlas_stream_connection_failover.test "yourWorkspaceName-6117ac2fe2a3d04ed27a9871-KafkaConnection-671f9c2fe2a3d04ed27a9880"
```

For more information see: [MongoDB Atlas API - Streams Failover Connection](https://www.mongodb.com/docs/api/doc/atlas-admin-api-v2/group/endpoint-streams) documentation.
